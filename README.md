# Робот: управление с ПК

Программа на Python снимает веб-камеру, распознаёт руки (MediaPipe) и шлёт команды на машинку по Wi‑Fi (ESP8266 в режиме точки доступа). Дополнительно можно управлять с клавиатуры.

## Подключение

1. Загрузи прошивку из папки `firmware/esp32_car_drive_server/` на ESP8266.
2. На ESP8266 поднимается Wi‑Fi точка с именем из прошивки (по умолчанию `Make DIY`).
3. Подключи ноутбук к этой сети.
4. В Serial Monitor при старте ESP выводится IP точки. Чаще всего это **192.168.4.1**.
5. В `main.py` задай тот же адрес в переменной **`CAR_WIFI_BASE_URL`** (например `http://192.168.4.1`). Если ESP в другой сети — укажи его реальный IP.

## Установка (ПК)

```bash
pip install -r requirements.txt
```

Нужна модель MediaPipe: файл `models/hand_landmarker.task` в корне проекта (рядом с `main.py`).

Запуск:

```bash
python main.py
```

Окно **Image** должно быть **в фокусе**, иначе клавиши не сработают.

## Управление жестами (руки)

Кадр с камеры **зеркалится** по горизонтали: «левая» и «правая» рука — как ты их видишь на экране.

### Левая рука (управление газом, тормозом, назад)

Жест определяется по **4 пальцам** (указательный, средний, безымянный, мизинец):

| Жест | Пальцы вверх | Действие |
|------|-------------|----------|
| **Кулак** | ни одного | Стоп |
| **Открытая ладонь** | все 4 | Вперёд |
| **«Рога»** (указательный + мизинец) | 1-й и 4-й | **Назад** |
| Любая другая комбинация | — | Стоп |

«Рога» — это жест, когда указательный палец и мизинец подняты, а средний и безымянный сложены.

### Правая рука (поворот)

Нужна **левая рука в режиме «газ»** (открытая ладонь) и **не** в режиме «назад» (рука не слишком низко).

| Наклон ладони (угол в плоскости кадра) | Действие |
|----------------------------------------|----------|
| Прямо (мертвая зона около центра) | Вперёд |
| Наклон в одну сторону | Вперёд + влево |
| Наклон в другую сторону | Вперёд + вправо |

Чувствительность мёртвой зоны: **`STEER_DEAD_ZONE_DEG`** в `main.py`.

### Если правая рука не в кадре

При открытой левой ладони (и не «назад») машина едет **просто вперёд** без поворота.

## Управление с клавиатуры

Клавиатура **перекрывает** жесты, пока удерживается режим клавиш (короткий таймаут после последнего нажатия — см. **`_KEY_TIMEOUT_SEC`** в `main.py`).

| Клавиша | Действие |
|---------|----------|
| **W** или стрелка **вверх** | Вперёд |
| **S** или стрелка **вниз** | Назад |
| **A** или стрелка **влево** | Вперёд + влево |
| **D** или стрелка **вправо** | Вперёд + вправо |
| **Пробел** | Стоп |
| **Q** | Выход из программы |

Стрелки обрабатываются через **`cv2.waitKeyEx`** (на Windows обычный `waitKey` стрелки не отдаёт). Снова: фокус на окне с видео.

На HUD справа внизу: подсказка про клавиши или активный режим **KB: …**.

---

## Что видно на экране (HUD)

- Состояние левой и правой руки, угол наклона для руля.
- Текущая команда **CMD:** и буква состояния для машинки (**State=**), как в прошивке (`F`, `B`, `G`, `I`, `S`).
- Строка **WiFi:** — результат последнего HTTP-запроса к ESP (`ok`, `timeout`, `net err` и т.д.).

---

## Прошивка (кратко)

ESP8266 поднимает веб-сервер и принимает запросы вида:

`GET http://<IP>/?State=<буква>`

Буквы: `F` вперёд, `B` назад, `G` вперёд-влево, `I` вперёд-вправо, `S` стоп (и другие — см. `esp32_car_drive_server.ino`).

## Настройка под себя

| Параметр в `main.py` | Значение |
|----------------------|----------|
| `CAR_WIFI_BASE_URL` | URL ESP8266 |
| `STEER_DEAD_ZONE_DEG` | Мёртвая зона руля по углу (правая рука) |
| `_KEY_TIMEOUT_SEC` | Как долго после отпускания клавиш держится режим клавиатуры |

При проблемах с сетью сначала проверь IP в Serial Monitor ESP и строку **WiFi:** на HUD.
